## Scope (10 gated sub-steps)

This sprint does **both halves in one branch**: first produce a complete severity-ranked findings log, then fix every confirmed finding and re-verify. The split below keeps each step small and owner-gateable. Steps 1–7 are **find-only** (no code changes — discovery into the log); Steps 8–10 are **fix + re-verify + exit**. For each find-only step, commit the running findings list / candidate notes to `docs/code-reviews/s13-findings.md` (draft) and push, so every discovery step is reviewable in the open PR before the first fix lands.

1. **(13-0) Plan & baseline.** Confirm S10/S11/S12 are merged to `main`; branch from latest `main`. Capture the green build baseline (`typecheck`/`lint`/`build`). Inventory the live routes by shell — public (incl. `/live` + `/live/[id]`), auth, `(workspace)` (incl. `programming`, `elements/[slug]`×30), admin — and the 4 auth states (anon · pending/unapproved · approved · admin). Restate the 6 audit lenses (access-gate · data-layer · public-pages · workspace-pages · auth-flows · cross-cutting) and the 5 testing hats. **Gate:** owner confirms scope and accepts find-first-then-fix.
2. **(13-1) Copywriting hat — every page, every auth state.** Verbatim-check every visible string against the `docs/page-copy/` canon; flag drift, invented strings, brand-voice violations (charity tone, hype, slogans, filler, exclamation marks), `[placeholder]` tokens, and any stale proof numbers — proof must read **10 · 30 · 200+ · 267 · 120-day launch** everywhere (no "3 gates", "250+", "29 topics", etc.). Discovery only → into the findings log (commit + push the running draft).
3. **(13-2) UX hat — flows, CTAs, links, states.** Walk every page as a brand-new user: every CTA goes where it claims, every link resolves, every form has clear success/error/empty/loading states, no dead-ends or loops, mobile reads clean at 320px. Discovery only (commit + push the running draft).
4. **(13-3) Feature/functional hat — access-gate + data-layer + auth-flows.** Verify every gated route enforces session + `is_approved()` server-side (anon → login/404, pending → pending state, approved → unlocked); every `/admin/*` checks `is_admin()` server-side, non-admin → 404 (admin gating is enforced in `src/app/admin/layout.tsx` via `is_admin()` for the whole `/admin` group — confirm any new S11 `/admin/content` page inherits the layout gate; absence of an inline per-page check is **not** a finding if the layout covers it); the full Apply→pending→approve→unlock→sign-out lifecycle works; templates/booklets download via server-issued signed URLs (raw paths never in source); `/build` progress persists; `(workspace)/programming` writes are owner-scoped; the S7-fixed auth defects stay fixed (no `/apply` enumeration oracle, no `/account` change-password dead-end, no silent `/update-password` bounce, signed-in users redirected off `/login` and `/apply`). Discovery only (commit + push the running draft).
5. **(13-4) a11y hat — WCAG AA.** Every page at 320px + desktop + reduced-motion: contrast (4.5:1 body, 3:1 large), never color-alone (errors/badges/locks pair color + text/icon), full keyboard nav with visible focus + reachable skip-link, heading order, semantic landmarks (single `<main>`), alt text, `prefers-reduced-motion` respected. Discovery only (commit + push the running draft).
6. **(13-5) Security hat — §15 invariants + headers + secrets.** Re-confirm the seven §15 invariants; CSP frame-src is YouTube-only (S9); headers present (HSTS, X-Frame DENY, nosniff, Referrer-Policy, Permissions-Policy); no secret in the client bundle or network tab (only `NEXT_PUBLIC_*`); no `dangerouslySetInnerHTML` fed user/query data; element bodies render only through the sanctioned MDX pipeline; no open redirect via `?next=`. Note the deferred rate-limit/Turnstile as the known S14 item, not a finding. Discovery only (commit + push the running draft).
7. **(13-6) Adversarial verify + produce the findings log.** For every candidate from 13-1…13-5, independently re-read the cited code and refute or confirm (strip false-positives), assign severity (High = gate breach / data leak / auth bypass · Medium = broken flow / dead-end / inconsistency · Low = cosmetic / typo / redundant · Nit = informational), and record reproduction + proposed fix. Write the complete log to `docs/code-reviews/s13-findings.md` in the S7 format (build baseline · severity summary table · HIGH/MEDIUM/LOW/NIT sections · verified-clean notes). **No fixes yet. Gate:** owner approves (a) the fix-priority order and (b) any new user-facing string wording — severity ranking is the engine's adversarial call, recorded in the log for owner visibility, not a per-item approval.
8. **(13-7) Fix-all.** Apply every confirmed fix in priority order (High → Medium → Low → Nit), **smallest safe change per fix**, gated in sensible batches. For any new user-facing string, draft per brand-voice and get owner approval at the gate before shipping; write approved copy to both the `.tsx` and the matching `docs/page-copy/` canon. Tick each finding `[x]` in the log. No new features, no redesign — correctness/copy/UX/a11y/security only.
9. **(13-8) Re-verify all fixes.** For every fix, independently confirm the repro is gone and no regression appeared; re-test the affected routes across the 4 auth states; `typecheck`/`lint`/`build` green after each batch; re-confirm the §15 invariants and the approval gate still hold after the fixes.
10. **(13-9) Exit gate.** Full-diff adversarial review of the whole S13 branch (security/gate · correctness · copy/design/a11y · build/config). Confirm the log shows every finding fixed + re-verified and zero known bugs remain. Update `docs/PROJECT-STATUS.md` (§1, §2 S13 row → ✅ with the findings-log link, §7 known-issues = only the S14 hardening item) and tick S13 in `docs/ROADMAP.md` §B. Owner runs the Preview + (post-merge) production smoke-test.
